# Команды по созданию ssh ключа

Описание

---

#### Создание и подключение SSH

```bash
ssh-keygen -o

# Инструкция далее:
# Нужно будет указать где сохранить файл (указывая пусть, можно сразу переименовать ключ)
# После нужно ввести пароль
# !Пароль должен быть сгенерирован!
```

#### Создание SSH ключа базовая команда

```bash
ssh-keygen

# Создаст ключи:
# id_rsa (приватный ключ)
# id_rsa.pub (публичный ключ)
```

#### Создание именованных SSH ключей

```bash
ssh-keygen -f ~/.ssh/github

# Создаст ключи:
# github (приватный)
# github.pub (публичный)
```

#### Создание ключа с указанием типа и имени

```bash
ssh-keygen -t rsa -b 4096 -f ~/.ssh/имя_ключа -C "комментарий"

# Примеры

# Для GitHub
ssh-keygen -t ed25519 -f ~/.ssh/github -C "your_email@example.com"

# Для рабочего сервера
ssh-keygen -t rsa -b 4096 -f ~/.ssh/work_server -C "work_access"

# Для личного использования
ssh-keygen -t ed25519 -f ~/.ssh/personal -C "personal_projects"
```

#### Полезные параметры ssh-keygen

```
-t - тип ключа (rsa, ed25519, ecdsa)
-b - длина ключа (для RSA)
-f - путь и имя файла
-C - комментарий (обычно email)
-N - парольная фраза
```

#### Проверка созданных ключей

```bash
# Посмотреть список ключей
ls -la ~/.ssh/

# Проверить отпечаток ключа
ssh-keygen -lf ~/.ssh/имя_ключа
```

#### Созданные ключи нужно добавить в SSH-агент командой

```bash
# Команда для доступа к ssh ключу
ssh-add <путь_до_публичного_ключе_без_расширения_.pub>

# Пример
ssh-add ~/.ssh/github
```
