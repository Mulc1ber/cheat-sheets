# Команды необходимые для минимальной работы с Git

Описание

---

> [Официальный сайт Git](https://git-scm.com/downloads)

### После скачивания и установки Git, можно ввести команду для проверки версии

- `git -v` или `git --version` - проверка установки git на ПК

### Инициализация Git в проекте

- `git init` - инициализирует git в папке проекта

### Настройки для коммитов

- `git config --global core.` - команда, которая позваляет узнать какой редактор откроется для коммита

- `git config --global core.editor "code --wait -n"` - команда, которая позволяет установить нужный для разраба редактор, в котором будет совершаться коммит.

### Если проект нужно клонировать с репозитория (GitHub / GitLab)

`git clone <url_repository>`

### Проверка привязки проекта к репозиторию

`git remote -v`

Если команда ничего не вывела в консоль, значит проект пока не привязан к удаленному репозиторию
Иначе командная строка выведет примерно следующее:

#### В случае клонирования с помощью HTTPS

```
origin https://github.com/<USER_NAME>/<REPO>.git (fetch)
origin https://github.com/<USER_NAME>/<REPO>.git (push)
```

#### В случае клонирования с помощью SSH

```
origin git@github.com:<USER_NAME>/<REPO>.git (fetch)
origin git@github.com:<USER_NAME>/<REPO>.git (push)
```

### Привязка проекта к удаленному репозиторию

`git remote add origin https://github.com/<USER_NAME>/<REPO>.git`
где `origin` - название ссылки к удаленному репозиторию.

### Удаление удаленного источника

`git remote rm <name_remote>`

### Параметры Git config

`git config user.name` и `git config user.email`
Данные параметры необходимы для указания автора комитов(commit) чтобы в Git можно было увидеть, кто делал конкретные изменения в проекте.

Для изменения или добавления данных, необходимо через пробел и в кавычках указать ИМЯ пользователя и его Email
`git config user.name "<USER_NAME>"`
`git config user.email "<EMAIl>"`

### Просмотр записи с комментариями в проекте

`git log`
Так же существует краткая запись комитов (commit):
`git log --oneline`

### Просмотр ветки

`git branch`

### Добавление ветки

`git branch <name_new_branch>`

### Удаление ветки

`git branch -d <name_branch>`

### Переключение (переход) на другую ветку

`git checkout <name_branch>`

### Создание с переходом на новую ветку

`git checkout -b <name_new_branch>`

### Удаление файлов из промежуточной области

`git reset <name_file>`

Чтобы добавить файлы в промежуточную область (**stage**), используют команду `git add <name_file>`

### Просмотр изменения в проекте (изменения, добавления, удаления)

`git diff`

### Удаления всех изменений, которые не были зафиксированы

`git reset --hard`

### Слияние веток

`git merge <name_branch>`

Перед слиянием веток необходимо перейти в ту ветку, в которую будут заливаться изменения. Далее прописываем команду и указывает имя ветки, которую хотим слить `git merge <name_branch>`.
